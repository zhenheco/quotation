# 前端錯誤診斷測試計劃

## 🎯 測試目標

根據用戶報告的問題：
1. **畫面滑動時出現短暫空白**
2. **刪除鍵點擊後出現白畫面**

## 📋 測試步驟

### 步驟 1：打開 Chrome DevTools Console

**Windows/Linux**: 按 `F12` 或 `Ctrl+Shift+I`
**Mac**: 按 `Cmd+Option+I`

### 步驟 2：配置 Console

1. 點擊 Console 標籤
2. 點擊設定圖標 ⚙️
3. 勾選以下選項：
   - ✅ "Preserve log"（保留日誌）
   - ✅ "Show timestamps"（顯示時間戳）

### 步驟 3：測試滑動功能

1. 清空 Console（點擊 🚫 圖標或按 `Ctrl+L`）
2. 重新載入頁面（`F5` 或 `Cmd+R`）
3. 向上/向下滑動頁面
4. 觀察是否出現空白畫面
5. **如果出現空白**，檢查 Console 的錯誤訊息
6. **如果看到 ErrorBoundary 錯誤頁面**，記錄錯誤訊息

### 步驟 4：測試刪除功能

1. 清空 Console
2. 前往有刪除功能的頁面（客戶/產品/報價單）
3. 點擊刪除按鈕
4. 確認刪除
5. 觀察是否出現白畫面
6. **檢查 Console 的錯誤訊息**

### 步驟 5：檢查 Network 請求

1. 點擊 Network 標籤
2. 勾選 "Preserve log"
3. 執行刪除操作
4. 檢查是否有失敗的請求（紅色顯示）
5. 點擊失敗的請求，查看 Response

## 🔍 要記錄的信息

當發現錯誤時，請記錄：

1. **錯誤訊息**（Console 中的紅色文字）
2. **堆疊追蹤**（Stack Trace - 點擊錯誤展開）
3. **相關文件名和行號**
4. **截圖**（包含整個 DevTools 視窗）

## 📸 截圖示範

截圖應該包含：
- ✅ 完整的 Console 面板
- ✅ 錯誤訊息
- ✅ 堆疊追蹤
- ✅ 時間戳
- ✅ 瀏覽器的 URL

## ⚠️ 預期的 ErrorBoundary 行為

如果添加的 ErrorBoundary 正常工作：
- ❌ **不應該**看到完全空白的畫面
- ✅ **應該**看到一個錯誤頁面，顯示：
  - 錯誤圖標
  - "頁面發生錯誤"標題
  - 錯誤訊息
  - 詳細堆疊追蹤（可展開）
  - "重新載入頁面"和"返回上一頁"按鈕

## 🚀 快速排查步驟

如果看到錯誤，嘗試：

1. **清除快取並重新載入**
   - `Cmd+Shift+R` (Mac) 或 `Ctrl+Shift+R` (Windows)

2. **檢查 Console 中的具體錯誤**
   - TypeError: Cannot read properties of undefined → 缺少 null 檢查
   - SyntaxError → 代碼語法錯誤
   - ReferenceError → 未定義的變數
   - Network Error → API 請求失敗

3. **提供錯誤信息**
   - 複製整個錯誤訊息（包括堆疊追蹤）
   - 提供截圖
   - 說明重現步驟
